# Hermes란?

**Hermes**는 [Nous Research](https://nousresearch.com/)에서 만든 **AI 에이전트 <Term tip="특정 목적에 맞게 미리 구조를 짜놓은 소프트웨어 기반 틀">프레임워크</Term>**입니다.

쉽게 말하면, 터미널에서 "이 파일 요약해줘" 라고 입력하면 — Hermes가 알아서 처리해주는 AI 비서입니다. Gateway를 설정하면 Slack·Telegram 같은 메신저에서도 쓸 수 있습니다.

## 에이전트가 뭔가요?

일반 AI 챗봇(ChatGPT 등)은 질문에 답변만 합니다. 반면 **에이전트(Agent)**는 여러 단계를 스스로 계획하고 실행합니다.

| 일반 AI | AI 에이전트(Hermes) |
|---------|------|
| 질문 → 답변 | 목표 → 계획 → 실행 → 결과 |
| 도구 없음 | 검색, 파일 읽기, 코드 실행 등 가능 |
| 1회성 | 연속적인 작업 처리 |

## Hermes의 특징

- **메신저 연결**: Gateway 설정 시 Slack·Telegram 등 메신저에서 사용 가능 (선택)
- **확장 가능**: <Term tip="AI 에이전트에 외부 도구·데이터를 플러그인처럼 연결하는 표준 규격">MCP(Model Context Protocol)</Term>로 다양한 도구 연결
- **Model 자유 선택**: Claude, OpenAI GPT, 로컬 model 등 원하는 LLM 선택 가능
- **오픈소스 기반**: [Nous Research Portal](https://portal.nousresearch.com/)에서 관리

<Callout type="info" title="처음 들어보는 용어들이 많아도 괜찮아요">
지금은 "Slack에서 쓰는 AI 비서구나" 정도만 이해해도 충분합니다. 설치하면서 자연스럽게 익히게 됩니다.
</Callout>

## 어떻게 쓰나요?

**기본 방식 — CLI**
1. 터미널에서 `hermes` 명령어로 실행합니다
2. `이 파일 분석해줘` 처럼 자연어로 요청하면
3. Hermes가 도구를 사용해 작업을 수행하고 결과를 알려줍니다

**선택 방식 — 메신저 (Gateway)**

Gateway를 설정하면 Slack·Telegram 채널에 `@hermes 이 파일 분석해줘` 라고 입력하고 결과를 채널에서 바로 받을 수 있습니다.
