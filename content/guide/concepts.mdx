# 구성 개념 소개

설치 전에 Hermes가 어떻게 동작하는지 흐름부터 잡으면 이후 설정이 훨씬 쉬워집니다.

## 동작 흐름

<HermesFlow />

Hermes는 단순히 질문에 답하는 챗봇이 아닙니다. 요청이 들어오면 도구를 실행하고, 결과를 기억에 남기고, 반복 작업은 Skill로 만들어 다음에 재사용합니다.

## 핵심 구성 요소

### Gateway
외부에서 Hermes에게 메시지를 보낼 수 있게 해주는 입구입니다. 설정하면 Slack·Telegram 등 메신저에서 Hermes를 호출할 수 있습니다. CLI로도 직접 사용할 수 있어 필수 요소는 아닙니다.

### Cron
정해진 시간에 자동으로 작업을 시작하는 기능입니다. "매일 오전 9시에 리포트를 만들어줘" 같은 정기 자동화에 사용합니다.

### LLM (대형 언어 모델)
Hermes의 "두뇌" 역할. Claude(Anthropic), OpenAI GPT 등을 선택해서 사용합니다.

### Agent (에이전트)
Hermes의 본체입니다. LLM(AI 모델)을 사용해 목표를 이해하고, 필요한 도구를 골라 실행합니다.

### Tool (도구)
에이전트가 할 수 있는 행동입니다. 예: 웹 검색, 파일 읽기, 터미널 명령 실행.

### MCP (Model Context Protocol)
도구를 에이전트에 연결하는 표준 방식입니다. 마치 플러그인처럼 필요한 기능을 추가합니다.

### Session
현재 진행 중인 대화와 작업 맥락입니다. 대화가 시작될 때 생성되고, 끝나면 사라집니다. Hermes는 과거 session을 검색해 이전에 했던 작업이나 결정을 다시 찾아볼 수 있습니다. 일시적인 작업 공간이라고 이해하면 됩니다.

### Memory
Hermes가 세션이 끝나도 유지하는 기억입니다. 사용자 선호·작업 방식은 `USER.md`에, 배운 규칙과 환경 정보는 `MEMORY.md`에 저장됩니다. 쓸수록 내 업무 방식에 맞게 맞춰지는 구조입니다.

### Skill
반복 작업의 절차를 저장해두고 슬래시 커맨드(`/요약`, `/번역` 등)로 호출하는 재사용 가능한 작업 절차입니다. 에이전트가 작업 중 직접 skill을 만들거나 수정해 재사용할 수도 있습니다.
