# 간단한 작업 지시

이 절에서는 Hermes Agent와 직접 대화하며 기본적인 작동 방식을 체험합니다. 사람에게 부탁하듯 자연스럽게 말하면 됩니다.

<Callout type="info" title="실습은 Desktop 앱 · Slack 채널 기준입니다">
이 실습 가이드는 **Hermes Desktop 앱의 채팅창** 또는 **Slack 채널**에서 Hermes를 호출하는 방법을 위주로 설명합니다. 터미널(CLI)에서도 같은 메시지를 그대로 입력하면 동일하게 동작합니다.
</Callout>

## 실습 환경

이 실습은 [AI 제공자 설정](/guide/prerequisites/api-keys)에서 골라 둔 Nous Portal의 무료 모델 `stepfun/step-3.7-flash:free`를 기준으로 합니다. 새로 가입해 크레딧이 없는 상태에서도 그대로 따라 할 수 있도록 한 선택입니다. 다른 모델을 쓰고 있다면 `/model`로 전환할 수 있고, 실습 결과는 모델에 따라 달라질 수 있습니다.

<Callout type="warning" title="무료 모델에는 민감한 정보를 넣지 마세요">
무료 모델은 속도 제한이 있고, 제공자에 따라 입력·출력이 품질 개선 목적으로 로깅될 수 있습니다. 시험·학습 용도로만 사용하고 개인정보·기밀 정보·업무 데이터는 입력하지 마세요. 실제 업무에는 유료 모델이나 로컬 LLM을 사용하는 것을 권장합니다.
</Callout>

## 실습 1: 말 걸어 보기

Desktop 앱을 실행하거나 Slack 채널에서 Hermes를 호출한 뒤, 편하게 말을 걸어 봅니다.

**Desktop 앱** — 채팅창에 그대로 입력합니다.

<TerminalBlock>
```
안녕, 너는 뭘 할 수 있어?
```
</TerminalBlock>

**Slack** — 채널에서 봇을 멘션해 입력합니다.

<TerminalBlock>
```
@Hermes 안녕, 너는 뭘 할 수 있어?
```
</TerminalBlock>

에이전트가 잠시 생각한 뒤, 자신이 할 수 있는 일을 자연스럽게 설명합니다. 파일 작업·터미널·웹 검색 같은 기본 도구와 설치 직후 활성화된 번들 스킬이 함께 소개됩니다. MCP 서버를 등록하거나 새 스킬을 추가하면 이 목록은 그만큼 풍성해지고, 같은 질문에도 답의 모양이 달라집니다. 단순한 챗봇이 아니라 실제로 행동하는 에이전트라는 것을 첫 대화에서 바로 느낄 수 있습니다.

## 실습 2: 정보 조사

에이전트에게 무언가를 조사해 달라고 부탁해 봅니다.

**Desktop 앱**

<TerminalBlock>
```
이번 주 화제가 된 오픈소스 프로젝트 하나를 골라서 어떤 프로젝트인지 요약해줘
```
</TerminalBlock>

**Slack**

<TerminalBlock>
```
@Hermes 이번 주 화제가 된 오픈소스 프로젝트 하나를 골라서 어떤 프로젝트인지 요약해줘
```
</TerminalBlock>

에이전트가 웹 검색 도구를 사용해 정보를 찾고, 요약해 줍니다. 직접 여러 페이지를 열어 읽는 대신 에이전트에게 맡기면 한 번에 정리된 답을 받을 수 있습니다.

<Callout type="tip" title="웹 검색이 안 된다면">
모델에 따라 웹 검색 결과를 도구 호출로 깔끔히 처리하지 못하는 경우가 있습니다. 무료 모델이나 14B 이하의 로컬 LLM에서 자주 나타납니다. 이때는 "검색하지 말고 네가 아는 대로 알려줘"라고 하면 학습 데이터를 기반으로 답합니다. 웹 검색은 모델의 도구 호출 능력에 크게 의존하며, 14B 이상 모델이나 프론티어 클라우드 모델에서 더 안정적입니다.
</Callout>

## 실습 3: 작업 맡기기

이번에는 단순한 질문이 아니라 결과물을 만들어 달라고 부탁합니다.

**Desktop 앱**

<TerminalBlock>
```
처음 텃밭을 시작하는 사람을 위한 봄철 채소 재배 가이드를 만들어서 파일로 저장해줘
```
</TerminalBlock>

에이전트가 작업하는 동안 Desktop 앱에는 **Thinking**, **Read file**, **Edit file** 같은 항목이 토글로 표시됩니다. 토글을 펼치면 에이전트가 어떤 도구를 어떤 순서로 쓰는지 직접 볼 수 있습니다.

**Slack**

<TerminalBlock>
```
@Hermes 처음 텃밭을 시작하는 사람을 위한 봄철 채소 재배 가이드를 만들어서 파일로 저장해줘
```
</TerminalBlock>

<Callout type="info" title="Slack에서는 진행 과정이 보이지 않습니다">
Slack에서는 도구 사용 진행 상황이 토글로 표시되지 않고 최종 결과만 전달됩니다. 에이전트가 어떤 도구를 어떻게 썼는지 단계별로 보고 싶다면 Desktop 앱을 사용하세요.
</Callout>

에이전트가 스스로 판단해 필요한 도구를 고르고, 계획을 세우고, 파일로 저장합니다. 웹 검색이 필요하다고 판단하면 알아서 검색하고, 아니면 바로 작성에 들어갑니다. 일반 챗봇은 텍스트를 보여주기만 하지만, Hermes Agent는 도구를 조합해 실제로 파일을 생성합니다.

이어서 수정을 요청해 봅니다.

**Desktop 앱**

<TerminalBlock>
```
방금 만든 봄철 채소 재배 가이드에서, 베란다 화분에서도 키울 수 있는 채소 위주로 다시 정리하고 물 주는 주기도 표로 넣어줘
```
</TerminalBlock>

**Slack**

<TerminalBlock>
```
@Hermes 방금 만든 봄철 채소 재배 가이드에서, 베란다 화분에서도 키울 수 있는 채소 위주로 다시 정리하고 물 주는 주기도 표로 넣어줘
```
</TerminalBlock>

에이전트가 직전에 만든 파일을 기억하고 있다면, 자세히 말하지 않아도 그 파일을 이어서 고칩니다.

<Callout type="info" title="모델에 따라 새 파일을 만들 수도 있습니다">
모델에 따라 직전 작업을 충분히 기억하지 못하고 파일을 새로 만들어 버리는 경우가 있습니다. 이를 줄이려면 위 예시처럼 **방금 만든 문서가 무엇인지 요청문에 직접 언급**하는 것이 안전합니다.
</Callout>

같은 질문을 다른 모델로 던져 보고 싶을 때는 모델을 바꿔 비교할 수 있습니다.

- **Desktop 앱** — 채팅 입력창 오른쪽의 모델 드롭다운에서 원하는 모델을 선택합니다.
- **Slack** — 모델을 직접 고르는 메뉴는 없습니다. 굳이 바꾸려면 `@Hermes 모델을 ~로 바꿔줘`처럼 요청하면 됩니다.

## 관찰 포인트

첫 대화에서 주목할 점들:

- **자연어 이해**: 딱딱한 명령어가 아니라 일상적인 말로 지시해도 의도를 파악합니다.
- **도구 선택**: 실습 3에서 Desktop 앱의 토글로 확인한 것처럼, 에이전트가 어떤 도구를 어떤 순서로 쓸지 스스로 판단합니다. 검색이 필요하면 검색하고, 파일을 써야 하면 파일을 씁니다.
- **컨텍스트 유지**: "베란다 화분 위주로 다시 정리해줘"처럼, 앞선 대화를 기억하고 있는지 확인하세요.
- **챗봇과의 차이**: 단순히 텍스트를 생성하는 것이 아니라, 도구를 조합해 파일을 저장하고 명령을 실행하는 등 실제로 행동한다는 점을 체감하세요.
