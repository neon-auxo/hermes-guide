# 뉴스 브리핑 자동화

이 절에서는 [반복 작업](/guide/practice/recurring-tasks)에 **웹 검색**과 **Slack 전달**을 결합해, 정해진 시간마다 관심 주제의 뉴스를 요약해서 Slack으로 받는 시스템을 만들어 봅니다. 지정한 시간이 되면 Slack에 그날의 브리핑이 도착합니다.

## 목표

- 매일 지정한 시간에 특정 주제의 뉴스를 검색한다
- 결과를 요약해 마크다운으로 정리한다
- Slack으로 전송한다

<Callout type="warning" title="먼저 확인할 것">
- **게이트웨이**가 켜져 있어야 반복 작업이 자동으로 돕니다.
- Slack으로 받으려면 받을 채널에서 `/sethome`을 한 번 해 둬야 합니다(설치 때 했다면 그대로 사용).
- 무료 모델은 웹 검색 결과를 도구로 깔끔히 정리하지 못할 때가 있습니다. 브리핑이 비거나 엉성하면 더 큰 모델로 바꿔 보세요.
</Callout>

## 1단계: 웹 검색 준비

뉴스 검색에는 Hermes에 **내장된 `web_search` 도구**를 씁니다. 별도 MCP 서버를 붙일 필요 없이 바로 쓸 수 있고, 백엔드로 Brave Search·Tavily·SearXNG·DuckDuckGo 등을 고를 수 있습니다.

<Callout type="info" title="키 없이도 동작하지만, 키를 넣으면 더 좋습니다">
검색 API 키 없이도 기본 백엔드로 동작하지만, 결과 품질과 안정성을 위해 검색 API 키를 넣어 두는 것을 권장합니다.
</Callout>

예를 들어 Brave Search를 쓰려면 Desktop 앱에서 키를 넣습니다.

1. 우측 상단 **설정 아이콘** 클릭
2. **Tools & Keys → Tools** 이동
3. **BRAVE SEARCH** 항목에 발급받은 API 키 입력

<Callout type="info" title="키는 바로 반영됩니다 (안 되면 재시작)">
입력한 키는 `~/.hermes/.env`에 저장되며 실행 중인 앱에 즉시 반영됩니다. 혹시 검색에 적용되지 않으면 앱(또는 게이트웨이)을 재시작하세요.
</Callout>

<Callout type="info" title="무료 검색 API">
[Brave Search API](https://brave.com/search/api/)는 무료 한도(월 약 2,000회)가 있어, 하루 한두 번 도는 일일 브리핑 용도로는 충분합니다.
</Callout>

## 2단계: 브리핑 작업 등록

이 실습에서는 **Slack 채널에서 `@Hermes`를 멘션해** 작업을 등록합니다. 그러면 결과도 그 채널로 돌아옵니다. 여기서는 예시 주제로 **"AI 비즈니스 활용 동향"**을 씁니다(관심 분야로 바꿔도 됩니다).

<TerminalBlock>
```text
@Hermes 매일 오후 3시에 "AI 비즈니스 활용 동향" 관련 최신 뉴스를 검색하고,
핵심을 한국어 3줄로 요약해서 Slack으로 보내줘.
```
</TerminalBlock>

크론에는 별도의 자연어 파서가 있는 게 아닙니다. **LLM이 자연어를 해석해** `0 15 * * *`(매일 오후 3시) 같은 형식으로 바꿔 등록합니다. 위 요청은 대략 "매일 오후 3시 → `web_search`로 주제 검색 → 상위 기사 선별 → 한국어 3줄 요약 → Slack 전송" 작업이 됩니다.

등록됐는지 확인합니다.

<TerminalBlock>
```text
@Hermes 지금 등록된 반복 작업 목록을 보여줘
```
</TerminalBlock>

<Callout type="info" title="Desktop 앱에서도 확인·편집할 수 있습니다">
Desktop 앱에서는 좌측 하단 **Cron**을 클릭하거나, 좌측 메뉴의 **Cron Jobs**에서 등록된 스케줄을 선택해 내용을 보고 **Edit cron**으로 고칠 수 있습니다.
</Callout>

### 브리핑 메시지 예시

Slack으로 받는 메시지는 대략 이런 형태입니다.

<TerminalBlock title="Slack 메시지">
```markdown
📊 경영 인사이트 브리핑 (2026-06-25)

1. 생성형 AI, 기업 도입 본격화
   - 사내 업무에 AI 비서를 도입하는 기업이 늘어나는 추세
   - 도입 효과 측정과 거버넌스가 새로운 과제로

2. 글로벌 금리·환율 동향
   - 주요국 통화정책 방향에 시장의 관심 집중
   - 수출 기업의 환헤지 전략 재점검 필요성 대두

3. 리더십: AI 시대의 의사결정
   - 데이터 기반 판단과 경험적 직관의 균형이 화두
   - 조직의 학습 속도가 경쟁력으로 부각
```
</TerminalBlock>

## 3단계: 형식 다듬기

브리핑이 매일 같은 틀로 오도록 출력 형식을 구체적으로 지정하면 읽기 편합니다.

<TerminalBlock>
```text
@Hermes AI 비즈니스 동향 브리핑 작업을 이렇게 바꿔줘.
- 맨 위에 오늘 날짜와 한 줄 헤드라인
- 소식마다 "제목 / 한 줄 요약 / 출처 링크"
- 마지막에 "오늘의 시사점" 한 줄
- 전체를 마크다운으로
```
</TerminalBlock>

<Callout type="tip" title="구체적으로 지시할수록 일관됩니다">
"뉴스 정리해줘"보다 위처럼 **항목·순서·형식**을 정해 주면 매일 결과가 비슷한 틀로 나옵니다.
</Callout>

## 4단계: 여러 주제를 다른 시간에

주제별로 시간을 나눠 여러 브리핑을 둘 수 있습니다.

<TerminalBlock>
```text
@Hermes 아래 세 가지 브리핑을 각각 등록해줘.
- 매일 오후 3시: "AI 비즈니스 활용 동향" 브리핑
- 매일 오후 3시 30분: "글로벌 경제·금융 동향" 브리핑
- 매주 금요일 오후 6시: "이번 주 경영·리더십 인사이트" 주간 요약
```
</TerminalBlock>

주제가 많아질수록 실행 시간과 모델 호출 비용이 늘어납니다. 처음에는 **주제 1~2개**로 시작해 보세요.

## 5단계 (고급): 새 소식만, 중복 없이

매일 억지로 채운 브리핑보다 **새로운 소식이 있을 때만** 받는 편이 깔끔합니다. [반복 작업](/guide/practice/recurring-tasks)에서 배운 `[SILENT]` 패턴과 **메모리**를 함께 씁니다.

<TerminalBlock>
```text
@Hermes 주중 30분마다 "AI 비즈니스 활용 동향" 키워드로 뉴스를 검색하고,
새로운 기사가 있으면 즉시 Slack으로 알려줘. 이전에 보낸 기사는 다시 보내지 마.
새 기사가 없으면 [SILENT]로 시작하는 짧은 응답만 작성해줘.
```
</TerminalBlock>

이렇게 하면 에이전트가 이미 보낸 기사 주소를 **메모리에 기록**해 두고 중복을 거릅니다. `[SILENT]`로 시작하는 결과는 전달되지 않으므로, 새 소식이 없는 동안에는 알림이 오지 않습니다.

## 메모리 활용

크론 작업에서 **메모리**를 활용하면 실행 사이에 상태를 유지할 수 있습니다. "이전에 전송한 기사 목록을 메모리에 저장해줘"처럼 지시하면, 에이전트가 `~/.hermes/memories/` 아래에 상태 파일을 만들어 관리합니다. 각 크론 작업은 매번 새 세션에서 따로 돌기 때문에, 이렇게 파일로 남겨 둬야 다음 실행이 이전 결과를 참고할 수 있습니다.

## 확장 아이디어

- **RSS 모니터링**: 특정 블로그·매체의 RSS를 주기적으로 확인해 새 글만 요약
- **주간 종합 리포트**: 일일 브리핑을 모아 주간 트렌드로 정리
- **멀티 채널 전송**: 같은 브리핑을 Slack과 다른 채널에 함께 전송
- **톤 분석**: 뉴스의 긍정·부정 분위기를 함께 요약해 시장 분위기 파악

## 문제 해결

**검색 결과가 없을 때** — 키워드가 너무 좁으면 결과가 비어 옵니다. 범위를 조금 넓혀 보세요.

<TerminalBlock>
```text
# 너무 구체적
"Hermes Agent 2.1 릴리스 노트"

# 적절한 범위
"AI 에이전트 동향"
```
</TerminalBlock>

**브리핑이 오지 않을 때**

- 게이트웨이가 실행 중인지 확인합니다(Desktop 앱 좌측 하단 **Gateway ready**).
- Slack에서 `@Hermes 지금 등록된 반복 작업 목록을 보여줘`로 작업이 살아 있는지 확인합니다.
- Slack 홈 채널(`/sethome`)이 지정돼 있는지 확인합니다.
- 무료 모델이라면 검색·요약이 불안정할 수 있으니 더 큰 모델로 바꿔 봅니다.
