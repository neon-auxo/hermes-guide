# 스킬 만들기

이 절에서는 **스킬(Skill)**이 무엇인지 간단히 익힌 뒤, 직접 스킬을 만들어 실행하고, 에이전트가 스킬을 자동 생성하는 과정도 관찰합니다.

## 스킬이란?

스킬은 `~/.hermes/skills/` 폴더에 저장되는 **"이런 작업은 이렇게 해라"는 절차서**입니다. 사람이 읽을 수 있는 마크다운 문서이며, [agentskills.io](https://agentskills.io) 오픈 표준을 따릅니다.

- 각 스킬은 하나의 폴더이고, 그 안에 최소한 **`SKILL.md`** 파일이 들어 있습니다. 필요하면 `scripts/`·`templates/`·`references/`·`assets/` 같은 하위 폴더가 함께 들어갑니다.
- 출처는 세 갈래입니다. **번들**(설치 시 함께 제공) · **Skills Hub 설치**(`hermes skills install`) · **에이전트 자동 생성**(반복 작업을 스킬로 저장).
- 설치된 모든 스킬은 자동으로 **`/스킬이름`** 슬래시 명령이 됩니다.

<Callout type="info" title="스킬은 필요할 때만 펼쳐집니다 (단계적 로딩)">
스킬은 토큰을 아끼려고 단계적으로 로딩됩니다. 시작 시에는 **이름과 설명만** 시스템 프롬프트에 들어가고(스킬 하나당 아주 짧은 분량), 에이전트가 필요하다고 판단하면 그때 목록과 전체 내용을 불러옵니다. 그래서 스킬이 수십 개여도 가볍습니다.
</Callout>

<Callout type="warning" title="핵심은 description">
에이전트는 각 스킬의 **`description`**을 보고 "지금 이 스킬을 쓸지"를 판단합니다. 그래서 description에 **어떤 상황에서 쓰는 스킬인지와 불러낼 만한 표현**(예: "이메일로 써줘")을 구체적으로 적는 것이 가장 중요합니다.
</Callout>

## 실습 1: 설치된 스킬 보기

먼저 지금 어떤 스킬이 있는지 봅니다. Desktop 앱에서는 좌측 **Skills & Tools** 메뉴에서 설치된 스킬을 살펴볼 수 있습니다.

CLI 채팅창에서 바로 확인하려면 아래를 입력하세요.

<TerminalBlock>
```
/skills
```
</TerminalBlock>

번들 스킬과 그동안 추가된 스킬이 함께 보입니다. 실제 파일이 어디 있는지는 `~/.hermes/skills/` 폴더를 직접 열어 확인할 수도 있습니다.

## 실습 2: 스킬 직접 만들기

가장 쉬운 방법은 **에이전트에게 만들어 달라고 부탁**하는 것입니다. 절차를 알려주면 에이전트가 `SKILL.md`를 대신 작성해 줍니다. "정중한 이메일로 다듬어 주는 스킬"을 만들어 봅니다.

<TerminalBlock>
```text
아래 절차의 스킬을 만들어줘.
이름: polite-email
1. 거친 메모를 받으면 핵심 용건을 한 문장으로 추린다
2. 정중한 한국어 이메일 형식(인사 → 용건 → 부탁 → 맺음)으로 쓴다
3. 제목 한 줄과 본문을 함께 출력한다
```
</TerminalBlock>

에이전트가 `~/.hermes/skills/polite-email/SKILL.md`를 생성합니다.

<Callout type="info" title="스킬 추가는 승인이 필요할 수 있습니다">
`SKILL.md`는 분량이 커서 바로 적용되지 않고 **검토 대기(승인) 상태**로 들어갈 수 있습니다. 내용을 확인하고 승인하면 스킬 목록에 추가됩니다.
</Callout>

추가됐다면 Desktop 앱 좌측 **Skills & Tools** 메뉴에서 우측 상단 **Search skills**로 `polite-email`을 검색해, 목록에 들어와 있는지 확인할 수 있습니다.

확인했다면 **새 세션을 열고** 슬래시 명령으로 실행해 봅니다. (새로 추가한 스킬은 새 세션부터 슬래시 명령으로 잡힙니다.)

<TerminalBlock>
```
/polite-email 내일 회의 30분만 미루자, 자료 아직 준비 중이라 점심 지나서 보낼게
```
</TerminalBlock>

메모가 정중한 이메일 형식(제목 + 인사 → 용건 → 부탁 → 맺음)으로 정리되는지 확인하세요.

## 실습 3: 스킬 내용 확인·수정

스킬은 그냥 마크다운 파일이라, 직접 열어 보거나 고칠 수 있습니다. `~/.hermes/skills/polite-email/SKILL.md`를 열면 아래와 비슷한 구조입니다.

<TerminalBlock title="SKILL.md">
```markdown
---
name: polite-email
description: 거친 메모나 요점을 정중한 한국어 이메일로 바꾼다. 사용자가 "이메일로 써줘", "정중하게 메일로 다듬어줘"처럼 요청할 때 사용한다.
---

# 정중한 이메일 작성

## 언제 쓰는가
사용자가 요점·메모를 정중한 이메일 본문으로 만들어 달라고 할 때.

## 작성 절차
1. 핵심 용건을 한 문장으로 추린다
2. "안녕하세요," 인사로 시작한다
3. 용건 → 부탁(정중한 청유형) → 맺음 순서로 쓴다
4. "감사합니다."와 보내는 사람 자리를 남긴다

## 출력 형식
- 제목: 한 줄
- 본문: 인사 → 용건 → 부탁 → 맺음
```
</TerminalBlock>

맨 위 `---`로 감싼 설정 부분에서 **꼭 필요한 것은 `name`과 `description`** 두 가지입니다. 특히 `description`에 "언제 쓰는 스킬인지"와 불러낼 만한 표현을 함께 적어 두면, 사용자가 슬래시 명령을 직접 치지 않아도 에이전트가 알맞은 상황에서 스킬을 찾아 씁니다. 시험 삼아 슬래시 명령 없이 부탁해 보세요.

<TerminalBlock>
```
이 메모를 정중한 이메일로 써줘: 다음 주 워크숍 장소를 회의실 대신 카페로 바꾸고 싶어요
```
</TerminalBlock>

## 실습 4: 자동 스킬 생성 관찰

에이전트는 여러 도구를 조합한 작업을 끝낸 뒤, 그 과정을 스킬로 저장할지 **백그라운드로 스스로 검토**하기도 합니다. 도구 호출이 여러 번 쌓이는 작업을 시켜 봅니다.

<TerminalBlock>
```
이 폴더의 마크다운 파일을 모두 찾아서, 각 파일의 제목과 첫 문단을 모아 한 개의 목차 문서로 정리해줘
```
</TerminalBlock>

작업이 끝난 뒤 에이전트가 그 과정을 **스킬로 저장하고 알림으로 알려주기도 합니다.** 다만 **자동 생성이 항상 일어나지는 않습니다.** 무엇을 스킬로 남길지는 모델이 판단하기 때문입니다. 원하면 직접 부탁해도 됩니다.

<TerminalBlock>
```
방금 한 작업을 스킬로 만들어줘
```
</TerminalBlock>

<Callout type="warning" title="로컬 LLM에서는 수동 생성을 권장">
작은 로컬 모델(7B~13B대)은 어떤 스킬을 언제 쓸지, 무엇을 스킬로 남길지 판단이 흔들립니다. 불필요한 스킬을 만들거나 좋은 기회를 놓칠 수 있으니, 자동 생성에 기대기보다 직접 만들고 생성된 스킬을 검토·수정하는 편이 안전합니다.
</Callout>

## 스킬 출처와 신뢰 등급

Skills Hub에서 스킬을 받을 때는 출처에 따라 신뢰 등급이 다릅니다. 특히 **community** 등급은 누구나 올릴 수 있으므로, 설치 전 내용을 한 번 살펴보는 습관이 좋습니다.

| 등급 | 설명 |
|------|------|
| **builtin** | Hermes에 내장돼 함께 배포되는 스킬. 스캔 없이 신뢰됩니다. |
| **trusted** | 검증된 공식 저장소(예: `openai/skills`, `anthropics/skills`)에서 온 스킬 |
| **community** | 그 외 누구나 배포할 수 있는 스킬 |

Hub에서 설치하는 스킬은 데이터 유출·프롬프트 인젝션·공급망 공격 패턴을 점검하는 보안 스캔을 거칩니다. 그래도 community 스킬은 설치 전 `hermes skills inspect <이름>`으로 내용을 직접 확인하는 것을 권장합니다.

<Callout type="warning" title="관찰 포인트 — 스킬">
- 스킬은 `~/.hermes/skills/<이름>/SKILL.md` 폴더 하나입니다. 설치되면 `/이름` 슬래시 명령이 됩니다.
- 맨 위 `---` 설정 부분의 필수 항목은 `name`과 `description`. **description이 자동 선택을 좌우**합니다.
- 슬래시 명령 없이도, description이 잘 적혀 있으면 에이전트가 알맞은 상황에서 스스로 스킬을 씁니다.
- 자동 생성은 모델 판단에 달려 있어 항상 일어나지는 않습니다. 필요하면 "방금 작업을 스킬로 만들어줘"라고 직접 요청하세요.
</Callout>
