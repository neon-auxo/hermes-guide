# 반복 작업 만들기

이 절에서는 **정해진 시간에 에이전트가 스스로 작업을 반복 실행**하게 만드는 크론(cron) 기능을 익힙니다. 복잡한 설정 없이, 대화로 "언제 무엇을 하라"고 말하면 됩니다.

<Callout type="warning" title="먼저 확인: 게이트웨이가 켜져 있어야 합니다">
반복 작업은 **게이트웨이(gateway)가 실행 중일 때만** 자동으로 돌아갑니다. 설치 때 게이트웨이를 백그라운드 서비스로 켜 뒀다면 그대로 두면 됩니다. 게이트웨이가 꺼져 있으면 예약 시간이 와도 아무 일도 일어나지 않습니다.
</Callout>

<Callout type="info" title="Desktop 앱에서 게이트웨이 상태 확인">
Desktop 앱에서는 좌측 하단의 **Gateway ready**를 클릭해 게이트웨이 상태를 확인할 수 있습니다.
</Callout>

## 시간을 말하는 법

자연어로 "매일 아침 9시"처럼 말하면, 에이전트가 알아서 적절한 형식으로 바꿔 등록합니다. 내부적으로 쓰이는 형식은 네 가지입니다.

| 형식 | 예시 | 의미 |
|------|------|------|
| 기간 | `30m`, `2h`, `1d` | 지금부터 그 시간 뒤 **한 번** 실행 |
| 반복 간격 | `every 30m`, `every 2h` | 그 간격으로 **반복** |
| 크론 표현식 | `0 9 * * *` | 표준 5필드 cron (매일 9시) |
| 특정 시각 | `2026-12-01T09:00:00` | 그 시각에 **한 번** 실행 |

<Callout type="info" title="1분 단위까지만 됩니다">
게이트웨이는 60초마다 한 번씩 할 일을 점검합니다. 그래서 "30초마다" 같은 더 잦은 주기는 만들 수 없습니다.
</Callout>

## 실습 1: 첫 반복 작업 등록

채팅창에 자연어로 부탁해 봅니다. 주중 아침마다 그날의 원달러 환율을 파일로 기록해 두는 작업입니다.

<TerminalBlock>
```text
주중(공휴일 제외) 매일 아침 10시 40분에 오늘 날짜 원달러 환율을 ~/daily/ 폴더에
report-YYYY-MM-DD.md 형식으로 만들어줘. 공휴일은 스크립트에 공휴일 목록을
하드코딩하여 처리해줘.
```
</TerminalBlock>

에이전트가 반복 작업을 등록합니다. 지금 등록된 작업을 확인해 봅니다.

<TerminalBlock>
```text
지금 등록된 반복 작업 목록을 보여줘
```
</TerminalBlock>

<Callout type="info" title="Desktop 앱에서 등록된 작업 확인">
Desktop 앱에서는 좌측 하단의 **Cron**을 클릭하거나, 좌측 메뉴의 **Cron Jobs**에 등록된 스케줄을 선택해서도 등록된 반복 작업을 확인할 수 있습니다.
</Callout>

작업 정보는 다음 위치에 쌓입니다.

<TerminalBlock title="~/.hermes/cron/">
```text
~/.hermes/cron/
├── jobs.json              # 등록된 모든 반복 작업
└── output/
    └── <작업 ID>/
        └── <실행 시각>.md  # 각 실행의 기록(에이전트 응답)
```
</TerminalBlock>

이번 실습에서 만든 환율 리포트 같은 **실제 산출물**은 작업에서 지정한 `~/daily/` 폴더에 쌓입니다. 위 `~/.hermes/cron/` 폴더에는 작업 정의(`jobs.json`)와 각 실행의 기록만 남습니다.

## 실습 2: 결과를 메시지로 받기

기본적으로 결과는 위 폴더에 **파일로만** 저장됩니다. Slack이나 Telegram 같은 채널로 받고 싶다면 두 가지가 필요합니다.

1. 해당 채널에서 `/sethome`으로 결과를 받을 홈 채널을 지정해 둔다 (설치 때 진행했다면 그대로 사용).
2. 작업을 등록할 때 보낼 곳을 함께 말한다.

<TerminalBlock>
```text
매일 아침 8시에 ~/daily/ 폴더에 어제 추가된 파일 개수를 세서 Slack으로 알려줘
```
</TerminalBlock>

채팅으로 말하는 것 외에 다른 방법도 있습니다.

- **Desktop 앱에서 직접 설정**: 좌측 하단 **Cron** 클릭 → 팝업에서 **Edit cron** → **Prompt**에 할 일을 적고(예: `10시 40분 작업 완료 후 ~/daily/ 폴더에서 오늘자 작업 결과를 요약해서 전달해줘`), **Deliver to**에서 **Slack**을 선택합니다. (Slack은 게이트웨이에 연결돼 있을 때 목록에 나타납니다.)
- **Slack에서 등록**: Slack 채널에서 `@Hermes`를 멘션해 위와 같이 자연어로 요청하면, 그 채널이 등록 위치가 되어 결과가 같은 채널로 돌아옵니다.

<Callout type="info" title="어디서 등록하느냐에 따라 기본 전달 위치가 다릅니다">
Slack·Telegram 같은 채팅 채널에서 등록한 작업은 그 채널로 결과가 돌아갑니다. 반면 CLI(`hermes`)에서 등록한 작업은 파일로만 남습니다. CLI에서 만든 작업의 결과를 채널로 받고 싶다면 위처럼 "Slack으로 보내줘"라고 분명히 말하세요.
</Callout>

## 실습 3: 문제가 있을 때만 알림 받기

매번 "이상 없음" 메시지가 오면 피로합니다. **문제가 있을 때만** 알림이 오게 만들 수 있습니다. 핵심은 정상일 때 응답을 `[SILENT]`로 시작하게 시키는 것입니다. `[SILENT]`로 시작하는 결과는 어디로도 전달되지 않습니다.

<TerminalBlock>
```text
매시간 ~/daily 폴더의 파일 개수를 세줘.
50개 이하이면 [SILENT]로 시작하는 짧은 응답만 작성하고,
50개를 넘으면 정리가 필요하다는 경고 메시지를 보내줘.
```
</TerminalBlock>

이렇게 하면 평소에는 조용하다가, 기준을 넘겼을 때만 알림이 옵니다. 시스템 점검·재고 감시처럼 "평소엔 조용, 이상 시에만 보고" 패턴에 유용합니다.

## 실습 4: 작업 수정·삭제·정지

등록한 작업도 대화로 관리합니다.

<TerminalBlock>
```text
매일 아침 10시 40분 환율 보고 작업을 아침 11시로 바꿔줘
모든 반복 작업을 잠시 멈춰줘
모든 반복 작업을 삭제해줘
```
</TerminalBlock>

Slack에서도 `@Hermes`를 멘션한 뒤 위와 같이 요청하면 작업을 수정·정지·삭제할 수 있습니다.

## 알아두면 좋은 것

<Callout type="warning" title="반복 작업 운영 주의점">
- **게이트웨이 필수**: 게이트웨이가 꺼져 있으면 예약 시간이 와도 실행되지 않습니다.
- **독립 실행**: 각 작업은 매번 새 세션에서 따로 돕니다. 이전 실행 결과를 이어 쓰려면 **파일로 저장**하도록 지시하세요.
- **재귀 방지**: 반복 작업이 도는 동안에는 새 반복 작업을 등록할 수 없습니다(작업이 또 작업을 만드는 무한 등록 차단).
- **놓친 작업**: 게이트웨이가 꺼져 있던 동안의 작업을 한꺼번에 몰아서 실행하지 않습니다. 잠깐 놓친 경우에만 한 번 따라잡습니다.
- **비용**: 매 실행마다 모델을 호출하므로, 잦은 주기는 토큰(또는 컴퓨팅) 비용이 듭니다. 처음에는 **하루 1~2회**로 시작하세요.
- **보안**: 등록할 때 프롬프트가 보안 검사(인젝션·데이터 유출·숨은 문자)를 거칩니다.
</Callout>

<Callout type="tip" title="작업 설계 팁 — 구체적으로 지시하기">
"시스템 상태를 확인해줘"보다 "디스크·메모리 사용량과 상위 5개 프로세스를 확인해 마크다운 표로 정리해줘"처럼 **무엇을, 어떤 형식으로** 만들지 구체적으로 적을수록 결과가 일관됩니다.
</Callout>
