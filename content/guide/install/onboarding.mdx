# 초기 설정 & Slack 연결

설치가 완료됐으면 바로 실행해 보세요. 별도 설정 없이도 Nous Portal 무료 플랜으로 바로 시작할 수 있습니다.

---

## 1단계: hermes 첫 실행

터미널을 열고 아래 명령을 실행하세요.

<TerminalBlock title="Hermes 실행">
```bash
hermes
```
</TerminalBlock>

AI 제공자가 아직 설정되지 않은 상태이면 Hermes가 자동으로 **브라우저에 Nous Portal 가입/로그인 화면**을 엽니다.

---

## 2단계: Nous Portal 가입 / 로그인

브라우저에 **"Welcome to Nous Portal"** 창이 뜨면 아래 중 하나로 가입하세요.

| 방법 | 설명 |
|------|------|
| **Google** | 구글 계정으로 1클릭 가입 (가장 빠름) |
| **이메일** | 이메일 주소 입력 → 확인 메일 클릭 |
| **GitHub** | GitHub 계정 연동 |
| **Discord** | Discord 계정 연동 |

<Callout type="tip" title="처음이라면 Google 추천">
Google 계정이 있다면 **Google** 버튼을 누르는 게 가장 빠릅니다. 별도 비밀번호 없이 기존 계정으로 바로 가입됩니다.
</Callout>

로그인이 완료되면 브라우저 창이 닫히고 **터미널의 hermes 세션으로 자동 복귀**됩니다. Hermes가 Nous Portal을 AI 제공자로 설정하고, 모델·도구 목록이 포함된 시작 배너가 표시됩니다.

---

## 3단계: 무료 모델 확인 / 변경

로그인 후 hermes 세션 안에 있는 상태에서, 슬래시 명령어로 모델을 확인하거나 바꿀 수 있습니다.

```
/model
```

대화형 메뉴에서 모델 목록이 표시됩니다. Nous Portal Free 플랜에서 무료로 사용할 수 있는 모델은 아래와 같습니다.

| 모델 | 특징 |
|------|------|
| `stepfun/step-3.7-flash:free` | 빠른 응답, 일반 대화·코딩에 적합 |
| `nvidia/nemotron-3-ultra:free` | 고품질 추론, 복잡한 작업에 적합 |

`:free`가 붙은 모델이 무료 모델입니다.

<Callout type="info">
`/`를 입력하면 사용 가능한 모든 슬래시 명령어 목록이 자동완성으로 표시됩니다.
</Callout>

---

## 4단계: 첫 대화

모델이 설정됐으면 hermes 세션에 바로 입력하세요.

```
현재 디렉토리에 어떤 파일이 있는지 알려줘
```

Hermes가 답변하면 설정 완료입니다.

<Callout type="tip">
`hermes --tui`로 실행하면 마우스 선택과 모달 오버레이를 지원하는 더 나은 UI로 시작할 수 있습니다. `/`를 입력하면 슬래시 명령어 자동완성이 뜹니다.
</Callout>

<Callout type="warning" title="대화가 안 된다면">
`hermes doctor`를 실행하면 문제를 진단하고 해결 방법을 안내합니다.
</Callout>

## Slack 연결 (선택 사항)

CLI만 쓴다면 이 단계는 건너뛰어도 됩니다. Slack에서 Hermes를 사용하려면 아래를 따라주세요.

### Gateway 설정

[사전 준비 — Slack 설정](/guide/prerequisites/slack-setup)에서 발급받은 토큰 3개가 필요합니다.

<TerminalBlock title="Gateway 대화형 설정">
```bash
hermes gateway setup
```
</TerminalBlock>

또는 `~/.hermes/.env` 파일에 직접 입력할 수도 있습니다:

<TerminalBlock title="~/.hermes/.env">
```bash
SLACK_BOT_TOKEN=xoxb-여기에붙여넣기
SLACK_APP_TOKEN=xapp-여기에붙여넣기
SLACK_ALLOWED_USERS=U01ABC2DEF3
```
</TerminalBlock>

### Gateway 시작

<TerminalBlock title="Gateway 실행">
```bash
hermes gateway
```
</TerminalBlock>

Slack에서 DM 또는 초대한 채널에 멘션을 보내보세요:

```
@Hermes 안녕
```

Hermes가 응답하면 연결 성공입니다.

### 응답이 없을 때 점검 순서

`@Hermes 안녕`을 보냈는데 반응이 없다면:

1. Socket Mode가 ON인지 확인 (Slack 앱 설정 3단계)
2. `message.channels` / `message.groups` / `message.im` / `app_mention` 이벤트가 모두 추가됐는지 확인
3. `channels:history` / `groups:history` scope가 추가됐는지 확인
4. **Reinstall to Agents**를 실행했는지 확인 (scope·이벤트 변경 후 필수)
5. `/invite @Hermes`로 봇을 채널에 초대했는지 확인
6. DM에서 응답이 없다면 **Chat Tab**이 ON인지 확인 (Slack 설정 5단계)
7. `SLACK_ALLOWED_USERS`에 내 Member ID가 들어있는지 확인
