# Slack 설정

Hermes는 CLI만으로도 바로 쓸 수 있습니다. Slack 연결은 선택 사항으로, Gateway를 통해 Slack 채널에서 명령을 입력하고 결과를 받고 싶을 때 설정합니다.

<Callout type="info" title="Slack 계정이 없다면?">
[slack.com](https://slack.com)에서 무료로 가입하고 새 워크스페이스를 만드세요. 개인 테스트용 워크스페이스를 하나 따로 만드는 걸 추천합니다.
</Callout>

이 페이지를 마치면 준비물이 세 가지 생깁니다:

- `xoxb-`로 시작하는 **Bot Token**
- `xapp-`로 시작하는 **App Token**
- `U`로 시작하는 **내 Slack Member ID**

## 1단계: Slack 앱 생성

1. [api.slack.com/apps](https://api.slack.com/apps) 접속
2. **Create New App** 클릭
3. **From scratch** 선택
4. App Name에 `Hermes` (또는 원하는 이름) 입력
5. 워크스페이스 선택 후 **Create App** 클릭

## 2단계: Bot 권한(Scope) 설정

1. 생성한 App을 클릭한 후 왼쪽 메뉴의 **Features → OAuth & Permissions** 클릭
2. 범위(Scopes) > 봇 토큰 범위(Bot Token Scopes) 섹션에서 **OAuth 범위 추가** 클릭
3. 아래 권한을 모두 추가:

| 권한 | 용도 |
|------|------|
| `app_mentions:read` | @멘션 읽기 |
| `channels:history` | public 채널 메시지 읽기 |
| `channels:read` | 채널 목록 조회 |
| `groups:history` | private 채널 메시지 읽기 |
| `chat:write` | 메시지 보내기 |
| `files:read` | 파일 · 음성 메시지 읽기 |
| `files:write` | 파일 업로드 |
| `im:history` | DM 읽기 |
| `im:read` | DM 정보 조회 |
| `im:write` | DM 열기 |
| `users:read` | 사용자 정보 조회 |

<Callout type="warning" title="channels:history와 groups:history는 필수">
이 두 scope가 없으면 채널 메시지를 받지 못합니다. DM에서만 작동하고 채널에서 반응이 없다면 가장 먼저 확인하세요.
</Callout>

## 3단계: Socket Mode 활성화

Hermes는 **Socket Mode**로 Slack과 연결합니다. 서버에 공개 URL이 없어도 작동합니다.

1. 왼쪽 메뉴 **Settings → Socket Mode** 클릭
2. **Enable Socket Mode** 토글 ON
3. App-Level Token 생성 팝업이 나타납니다:
   - Token Name에 원하는 이름 입력 (예: `hermes-socket`, 이름은 무엇이든 무관)
   - **connections:write** 스코프 추가
   - **Generate** 클릭
4. `xapp-`로 시작하는 토큰 복사 → 안전하게 보관 (이것이 `SLACK_APP_TOKEN`)

<Callout type="info">
App-Level Token은 **Settings → Basic Information → App-Level Tokens**에서 언제든 다시 확인하거나 재발급받을 수 있습니다.
</Callout>

## 4단계: Event Subscriptions 설정

1. 왼쪽 메뉴 **Features → Event Subscriptions** 클릭
2. **Enable Events** 토글 ON
3. **Subscribe to bot events**에 아래 이벤트 모두 추가:

| 이벤트 | 용도 |
|--------|------|
| `message.im` | DM 수신 |
| `message.channels` | public 채널 메시지 수신 |
| `message.groups` | private 채널 메시지 수신 |
| `app_mention` | @멘션 수신 |

4. **Save Changes** 클릭

## 5단계: Chat Tab 활성화

이 단계를 건너뛰면 사용자가 DM을 보낼 때 "Sending messages to this app has been turned off" 오류가 납니다.

1. 왼쪽 메뉴 **Features → App Home** 클릭
2. **Show Tabs** 섹션에서 **Chat Tab** 토글 ON
3. **"Allow users to send Slash commands and messages from the chat tab"** 체크

## 6단계: 앱 설치 & Bot Token 발급

1. 왼쪽 메뉴 **Settings → Install App** 클릭
2. **Install to Workspace** 클릭
3. 권한 요청 화면에서 **허용** 클릭
4. **Bot User OAuth Token** 복사 (`xoxb-`로 시작) → 안전하게 보관 (이것이 `SLACK_BOT_TOKEN`)

<Callout type="warning" title="스코프나 이벤트를 변경했다면 반드시 Reinstall">
권한(scope)이나 이벤트를 추가한 뒤에는 **Reinstall to Agents**를 다시 눌러야 적용됩니다.
</Callout>

## 7단계: 내 Slack Member ID 확인

Hermes는 허용된 사용자 목록(Member ID)으로 보안을 관리합니다. **이 설정이 없으면 모든 메시지가 차단됩니다.**

1. Slack에서 내 이름 또는 프로필 사진 클릭
2. **프로필(View full profile)** 클릭
3. `⋮` (더보기) 버튼 클릭
4. **멤버 ID 복사(Copy member ID)** 선택

Member ID는 `U01ABC2DEF3` 형태입니다. 이것이 `SLACK_ALLOWED_USERS`에 입력할 값입니다.

## 8단계: 봇을 채널에 초대

1. Slack 앱에서 테스트할 채널 열기
2. 채널 메시지 입력창에 입력:

```
/invite @Hermes
```

<WindowsNote>
Windows에서도 Slack 앱 절차는 동일합니다. 브라우저에서 api.slack.com에 접속하면 됩니다.
</WindowsNote>

---

여기까지 완료되면 세 가지 값이 준비된 상태입니다.

<Callout type="warning" title="토큰과 Member ID를 안전하게 보관하세요">
아래 세 값은 Hermes 설치 후 설정에 사용합니다. 비밀번호처럼 취급하고 메모장이나 비밀번호 관리 앱에 저장해두세요.

- `SLACK_BOT_TOKEN` — `xoxb-`로 시작하는 Bot Token
- `SLACK_APP_TOKEN` — `xapp-`로 시작하는 App Token
- `SLACK_ALLOWED_USERS` — 내 Slack Member ID

카카오톡, GitHub, Slack 채널 등 공개된 곳에는 절대 붙여넣지 마세요.
</Callout>

## 동작 방식

| 상황 | 응답 방식 |
|------|-----------|
| DM | @멘션 없이 모든 메시지에 응답 |
| 채널 | `@Hermes 질문` 처럼 @멘션이 있을 때만 응답 |
| 채널 스레드 | 봇이 한 번 응답한 스레드에서는 @멘션 없이도 이어서 응답 |

## Slack을 쓸 때 알아두면 좋은 것

**장점**
- 요청과 결과가 스레드에 남아 맥락을 추적하기 쉽습니다.
- 팀원이 같은 채널에서 진행 상황을 함께 볼 수 있습니다.
- 채널별로 주제를 분리해 역할을 구분하기 좋습니다.

**제한**
- Slack 무료 플랜은 **90일치** 메시지만 검색 가능합니다. 오래된 대화 기록이 필요하다면 Pro 플랜이 필요합니다.
- 채널에서는 @멘션이 있을 때만 응답하므로 규칙을 지키면 채널이 불필요하게 채워지지 않습니다.
